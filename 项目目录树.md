## 主要模块说明

### 核心模块
- **app**: 主应用模块，包含 UI、设置、文件管理等功能
- **autojs**: JavaScript 运行时核心，提供脚本执行引擎和 API
- **automator**: 自动化操作引擎，基于无障碍服务的 UI 自动化

### 功能模块
- **common**: 通用工具类和共享资源，被其他模块依赖
- **paddleocr**: PaddleOCR 文字识别功能
- **codeeditor**: 代码编辑器，支持语法高亮和插件系统
- **inrt**: 模板应用，用于将脚本打包成独立 APK
- **apkbuilder**: APK 构建工具，用于打包脚本为 APK

### 配置模块
- **buildSrc**: Gradle 构建脚本，包含版本管理和工具函数
- **gradle/**: Gradle 配置和包装器

## 技术栈文件结构

### Java/Kotlin
- 所有 `.kt` 和 `.java` 文件位于各模块的 `src/main/java/` 目录

### JavaScript/TypeScript
- 位于 `autojs/src/main/js/` 目录
- `v6-api/`: v6 版本 API（TypeScript）
- `v7-api/`: v7 版本 API（TypeScript）
- 构建后生成的模块位于 `assets/v6modules/` 和 `assets/v7modules/`

### 资源文件
- 布局文件: `src/main/res/layout/`
- 图片资源: `src/main/res/drawable-*/`
- 值资源: `src/main/res/values/`
- 国际化: `src/main/res-i18n/values-*/`

### 原生代码
- C++ 代码: `paddleocr/src/main/cpp/`
- JNI 库: `paddleocr/src/main/jniLibs/`
- 二进制文件: `autojs/src/main/assets/binary/`

## 构建产物目录

### APK 输出
- Debug: `app/build/outputs/apk/v7/debug/`
- Release: `app/build/outputs/apk/v7/release/`

### 模块输出
- 各模块的 `build/` 目录包含编译产物

## 注意事项

1. **项目使用多模块架构**，各模块之间有依赖关系
2. **JavaScript 模块需要先编译**：运行 `./gradlew autojs:buildJsModule`
3. **文档需要单独构建**：运行 `./gradlew app:buildDocs`
4. **支持多语言**，国际化资源位于各模块的 `res-i18n/` 目录
5. **主要支持 arm64-v8a 架构**，其他架构可通过配置启用
