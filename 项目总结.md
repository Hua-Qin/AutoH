# AutoX.js v7 项目总结

## 项目简介

AutoX.js v7 是一个支持无障碍服务的 Android 平台 JavaScript 运行环境和开发环境。该项目是从 Auto.js 4.1 版本基础上发展而来，旨在提供类似 JsBox 和 Workflow 的自动化脚本开发体验。

### 核心特性

1. **基于无障碍服务的自动化操作** - 提供简单易用的自动操作函数
2. **悬浮窗录制和运行** - 支持脚本的录制和回放
3. **强大的选择器 API** - 提供对屏幕控件的查找、遍历、获取信息和操作
4. **JavaScript 开发环境** - 支持代码补全、变量重命名、代码格式化等功能
5. **界面开发支持** - 支持使用 E4X 编写界面，可打包为独立 APK
6. **Root 权限支持** - 提供更强大的屏幕点击、滑动和 Shell 命令功能
7. **图像处理功能** - 提供截取屏幕、保存截图、找色、找图等函数
8. **Tasker 插件集成** - 可作为 Tasker 插件使用
9. **界面分析工具** - 类似 Android Studio 的 LayoutInspector

### 版本特性（v7 新增）

- 基于 Material Design 3 的全新 UI 界面
- 支持 Shizuku 功能，可动态调试基于 Shizuku 的 API
- 引入 Node.js 引擎（Javet），支持运行大量 npm 软件包
- 使用 TypeScript 编写模块并添加类型声明
- 全新基于 Vue3 和 Jetpack Compose 的 UI 框架
- 完善 APP 打包功能和签名管理
- Rhino 升级至 v1.8.0 稳定版，支持更多 ES6+ 语法

## 编译和构建方法

### 环境要求

1. **Java 版本**: JDK 17 或更高版本
2. **Node.js**: 20+ 版本（用于编译 JavaScript 模块）
3. **Android SDK**: Android Studio 配置的 SDK
4. **Gradle**: 8.5.0（项目自带）
5. **操作系统**: Windows、macOS 或 Linux

### 编译步骤

#### 1. 首次构建前准备

首次构建前需要编译 JavaScript 模块：

```bash
./gradlew autojs:buildJsModule
```

此命令会：
- 编译 v6 API（TypeScript → JavaScript）
- 编译 v7 API（TypeScript → JavaScript）
- 生成模块文件到 `assets/v6modules/` 和 `assets/v7modules/`

**注意**：只需运行一次，若更改了模块代码需再次运行。

#### 2. 构建文档（可选）

```bash
./gradlew app:buildDocs
```

此命令会构建项目文档并复制到应用资源中。

#### 3. 本地安装调试版本

```bash
./gradlew app:buildDebugTemplateApp && ./gradlew app:assembleV7Debug && ./gradlew app:installV7Debug
```

或在 Windows PowerShell（< 7.0）中使用：

```bash
./gradlew app:buildDebugTemplateApp ; ./gradlew app:assembleV7Debug ; ./gradlew app:installV7Debug
```

**说明**：
- `buildDebugTemplateApp`: 构建调试版模板应用
- `assembleV7Debug`: 编译 v7 调试版本
- `installV7Debug`: 安装到连接的设备

生成的调试版本 APK 文件位于：`app/build/outputs/apk/v6/debug/`

#### 4. 本地编译发布版本

```bash
./gradlew app:buildTemplateApp && ./gradlew app:assembleV7
```

或在 Windows PowerShell 中：

```bash
./gradlew app:buildTemplateApp ; ./gradlew app:assembleV7
```

**说明**：
- `buildTemplateApp`: 构建发布版模板应用
- `assembleV7`: 编译 v7 发布版本

生成的是**未签名**的 APK 文件，位于：`app/build/outputs/apk/v6/release/`

#### 5. Android Studio 中编译

**运行调试版本**：
1. 先运行：`./gradlew app:buildDebugTemplateApp`
2. 在 Android Studio 中点击运行按钮

**编译发布版本并签名**：
1. 先运行：`./gradlew app:buildTemplateApp`
2. 在 Android Studio 菜单选择：Build → Generate Signed Bundle / APK...
3. 选择 APK → Next
4. 选择或新建证书 → Next
5. 选择 v7Release → Finish

生成的 APK 文件位于：`app/v7/release/`

## 打包和上线流程

### 打包脚本为 APK

AutoX.js 提供将脚本打包为独立 APK 的功能，用户可以将自己的脚本分发为独立应用。

#### 打包步骤

1. **准备脚本项目**
   - 在 AutoX.js 中创建项目
   - 编写脚本代码
   - 配置项目属性

2. **配置项目**
   - 设置应用名称
   - 配置图标
   - 设置版本号
   - 配置权限

3. **打包 APK**
   - 在 AutoX.js 中选择打包功能
   - 选择或创建签名证书
   - 配置打包选项
   - 开始打包

4. **获取 APK**
   - 打包完成后获得独立 APK
   - APK 包含脚本和运行时
   - 可独立安装运行

#### 打包配置

##### 签名管理

**使用默认签名**：
- 应用提供默认签名证书
- 位于 `apkbuilder/src/main/assets/autox-default.jks`
- 适用于测试和开发

**创建自定义签名**：
1. 在设置中选择签名管理
2. 创建新的密钥库（Keystore）
3. 设置密钥别名和密码
4. 保存签名配置

##### 权限配置

打包时可配置的权限：
- 无障碍服务权限
- 悬浮窗权限
- Root 权限（可选）
- Shizuku 权限（可选）
- 网络权限
- 存储权限
- 其他 Android 系统权限

##### 特殊配置

- **Shizuku 支持** - 打包的应用可使用 Shizuku API
- **Node.js 模块** - 支持打包 Node.js 依赖
- **资源打包** - 包含项目中的图片、音频等资源
- **UI 定制** - 定制应用的启动界面和设置

### 打包流程详解

#### 应用内打包流程

1. **项目配置**
   - 用户在项目设置中配置应用信息
   - 包括应用名称、包名、版本号等

2. **模板应用构建**
   - 构建模板应用（inrt 模块）
   - 将模板 APK 复制到资源目录

3. **脚本注入**
   - 将用户脚本注入到模板应用
   - 更新项目配置文件

4. **资源处理**
   - 处理项目中的资源文件
   - 更新应用图标和启动图

5. **权限配置**
   - 根据配置添加权限声明
   - 修改 AndroidManifest.xml

6. **APK 构建**
   - 使用 AAPT2 构建 APK
   - 进行资源编译和链接

7. **签名**
   - 使用指定证书签名 APK
   - 支持 V1、V2、V3 签名

8. **对齐**
   - 对齐 APK 优化性能
   - 使用 zipalign 工具

#### 命令行打包（开发者）

```bash
# 构建模板应用
./gradlew :inrt:assembleTemplateRelease

# 打包脚本（需要自定义脚本）
# 应用内打包流程由代码实现
```
