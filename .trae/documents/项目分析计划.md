# AutoX.js v7 项目分析计划

## 任务目标
全面分析 AutoX.js v7 项目，生成两个独立的 Markdown 文档：
1. 项目目录树（`项目目录树.md`）
2. 项目总结（`项目总结.md`）

## 实施步骤

### 步骤 1：项目整体结构分析
- 分析项目的根目录结构
- 识别所有主要模块和子模块
- 了解项目的构建配置文件
- 分析依赖关系和模块间的连接

### 步骤 2：详细目录树生成
- 生成完整的项目目录结构
- 包含所有主要目录和文件类型
- 按照模块层次组织
- 标注重要文件和目录的用途

### 步骤 3：项目信息收集
- **编程语言分析**：
  - 主要开发语言：Kotlin、Java
  - 脚本语言：JavaScript/TypeScript
  - 构建语言：Gradle (Kotlin DSL)

- **主要模块识别**：
  - app：主应用模块
  - autojs：JavaScript 运行时核心
  - automator：自动化操作引擎
  - common：通用工具和共享资源
  - inrt：模板应用（用于打包）
  - paddleocr：OCR 功能模块
  - codeeditor：代码编辑器
  - apkbuilder：APK 构建工具

- **技术栈分析**：
  - Android SDK：compileSdk, targetSdk
  - UI 框架：Jetpack Compose + Material Design 3
  - JavaScript 引擎：Rhino + Javet (Node.js)
  - 图像处理：OpenCV、MLKit
  - 网络请求：OkHttp、Retrofit
  - 异步处理：Kotlin Coroutines、RxJava

### 步骤 4：编译和构建流程分析
- 分析 Gradle 构建配置
- 识别编译命令和流程
- 分析 JavaScript 模块构建过程
- 了解 APK 打包流程

### 步骤 5：打包和上线流程分析
- 分析打包配置
- 识别签名配置
- 了解发布流程
- 分析版本管理

### 步骤 6：生成项目目录树文档
创建 `项目目录树.md` 文件，包含：
- 完整的项目目录结构
- 目录和文件的简要说明
- 模块划分说明

### 步骤 7：生成项目总结文档
创建 `项目总结.md` 文件，包含：
- 项目简介
- 使用的技术栈和编程语言
- 主要模块功能说明
- 编译和构建方法
- 打包和上线流程
- 项目特色和亮点

## 预期输出
两个独立的 Markdown 文档：
1. `项目目录树.md` - 详细的项目目录结构
2. `项目总结.md` - 项目概览和技术文档

## 注意事项
- 文档使用中文编写
- 保持文档清晰易读
- 确保信息准确完整
- 按照模块层次组织内容
